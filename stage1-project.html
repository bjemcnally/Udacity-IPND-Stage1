The Box Revisited

The Box Model (see here)

  • margin - (reddish) clears area around the border, completely transparent, the space between boxes
    (horizontal top/bottom only??)
  • border - (orange) goes around padding, inherited from color property of the box (not sure what this means)
    (border-width, all the way around, four sides)
  • padding - (green) clears area around content, affected by background color of the box, space inside the box that protects the content
    (all the way around, four sides)
  • content - (blue) the image or text that appears on the website, other layers protect it so that other items on the page don't overlap or sit too close
    (width and height can be defined to give it shape)

Size of element = sum(border, padding, content widths)

* {
    -webkit-box-sizing: border-box;
    -moz-box-sizing: border-box;
    -ms-box-sizing: border-box;
    box-sizing: border-box;
}

"This makes calculation of element size include border and padding so it is easier to make layouts", sets width of overall box so no matter how padding and border are tweaked, the size of the box is the same. Remember: margin is NOT included!
Need to add browser-specific prefixes to make sure it is interpreted properly (first 3 lines)

Instead of pixels, you can set size of box as percentage, so it changes with screen size (this seems like it would generally be a good idea, I will probaby default to this)

Can also set max-width for box so it resizes as necessary but will never be larger than box width

This does not change the layout of the page, only the sizes of the boxes and if/how they change as the browser window changes in size

Positioning Boxes

Flex box layout (great guide can be found here)

Flex box = flexible box, allows you to distribute space among items in a div

.classname {
    display: -webkit-flex;
    display: flex;
}

This is an example! I had to alter the margin, padding, etc. of this box so that there was space between this text and the code on the left

Note: 'classname' should be the class of the div that contains the divs you want next to each other (aka the child elements). Also: the -webkit-flex is required to ensure it displays properly in Safari

These are my notes from the link in the title of this section:

  • By default, flex items (children divs within parent divs) are laid out in the source order, the 'order' property can change this. e.x. for child div 'second' that was put first in the HTML add:

    .second {
        order: 2;
    }

  • Within parent div (container) children divs (items) default to rows that go left to right, but there are other options:

    .container {
        flex-direction: row; /* this is the default */
        flex-direction: row-reverse; /* this will go right to left */
        flex-direction: column; /* this is will go top to bottom */
        flex-direction: column-reverse; /* this is will go bottom to top */
    }

  • flex-grow allows you to distribute space in side the container amongst the items. When set equal to 1 for all items the space will be distributed equally, if all but one item has flex-grow:2, that item will be given twice as much space:

    .item {
        flex-grow: 2;
    }

  • flex-wrap will allow the items to wrap as necessary

    .container {
        flex-wrap: nowrap; /* single line */
        flex-wrap: wrap; /* multi-line */
        flex-wrap: wrap-reverse; /* multi-line (don't totally get this one) */
    }

  • justify-content defines the alignment along the main axis and distributes free space left over when all items on a line are a set size or have reached their max size. Can be set to flex-start (everything aligns to the left), flex-end (everything aligns to the right), center, space-between, or space-around (these last ones are self explanatory)

    .container {
        justify-content: center; /* this is one example */
    }

Code, Test, Refine

Always code, test, and refine before proceeding! This an iterative process!

  1. Look for natural boxes (start with structure, 'boxify')
  2. Look for repeated styles and semantic elements (add tags, i.e. headings, etc)
  3. Write HTML
  4. Apply styles from BIGGEST to smallest
  5. Fix things (try out other browsers, resize the window, repeats steps 3 and 4 until you're satisfied)
